# Docker Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** Containerize the job search pipeline and PostgreSQL database so the app runs identically in development and production with `docker compose`.

**Architecture:** Multi-stage Dockerfile (builder installs deps, runtime runs them), single `docker-compose.yml` with `app` + `db` services. Secrets injected at runtime via `.env` file — never baked into the image. App container is used on-demand via `docker compose run --rm app python cli.py ...`.

**Tech Stack:** Docker, Docker Compose v2, postgres:16-alpine, python:3.12-slim

## Task 3: Create `Dockerfile`

**Files:**
- Create: `Dockerfile`

- [x] **Step 1: Create `Dockerfile`**

```dockerfile
# Stage 1 — build dependencies
FROM python:3.12-slim AS builder

RUN apt-get update && apt-get install -y --no-install-recommends \
    gcc \
    libpq-dev \
    python3-dev \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /build
COPY requirements.txt .
RUN pip install --prefix=/install --no-warn-script-location -r requirements.txt

# Stage 2 — runtime image
FROM python:3.12-slim

RUN apt-get update && apt-get install -y --no-install-recommends \
    libpq5 \
    && rm -rf /var/lib/apt/lists/*

COPY --from=builder /install /usr/local
COPY . /app

WORKDIR /app
```

- [x] **Step 2: Build the image**


Expected: build completes with no errors. The builder stage installs compiled packages (asyncpg requires gcc + libpq-dev); the runtime stage contains none of those build tools.

## Task 4: Create `docker-compose.yml`

**Files:**
- Create: `docker-compose.yml`

- [x] **Step 1: Create `docker-compose.yml`**

```yaml
services:
  db:
    image: postgres:16-alpine
    env_file: .env
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready"]
      interval: 5s
      timeout: 5s
      retries: 5

  app:
    build: .
    env_file: .env
    depends_on:
      db:
        condition: service_healthy
    volumes:
      - ${RESUME_DIR}:/data
      - ./config:/app/config
    command: ["tail", "-f", "/dev/null"]

volumes:
  pgdata:
```

- [x] **Step 2: Copy `.env.example` to `.env` and set Docker values**

```bash
cp .env.example .env
```

Then open `.env` and set:
```
POSTGRES_HOST=db
RESUME_DIR=/absolute/path/to/your/resumes/folder
```

Set real values for `POSTGRES_USER` and `POSTGRES_PASSWORD` (e.g. `jobsearch` / `secret`) — Postgres uses these to create the database on first start, and the connection test in Step 4 requires them. API key placeholders can stay as-is.

- [x] **Step 3: Start the `db` service**

```bash
docker compose up db -d
```

Expected: container starts. Wait ~5 seconds for healthcheck to pass, then:

```bash
docker compose ps
```

Expected: `db` shows status `healthy`.

- [x] **Step 4: Verify `app` can reach the database**

```bash
docker compose run --rm app python -c "
import asyncio, asyncpg, os
async def check():
    conn = await asyncpg.connect(
        host=os.environ['POSTGRES_HOST'],
        port=int(os.environ.get('POSTGRES_PORT', 5432)),
        database=os.environ['POSTGRES_DB'],
        user=os.environ['POSTGRES_USER'],
        password=os.environ['POSTGRES_PASSWORD'],
    )
    print('connected:', await conn.fetchval('SELECT version()'))
    await conn.close()
asyncio.run(check())
"
```

Expected: prints a Postgres version string.

- [x] **Step 5: Verify CLI is reachable inside compose**

```bash
docker compose run --rm app python cli.py --help
```

Expected: argparse help message listing `search`, `full`, `contacts-only` modes.

- [x] **Step 6: Stop services**

```bash

```
